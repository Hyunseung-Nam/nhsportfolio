1) 매장 운영 CRM 시스템 (포인트 관리)

Tags: Delivered · In Use · Maintained

한 줄 요약: 매장 운영자가 고객에게 포인트를 보다 편하게 지급할 수 있도록, 운영 편의성 개선 요청을 기준으로 CRM을 설계·구현했습니다.

배경(요청): 특정 운영상의 문제를 전제로 하기보다는, 매장에서 고객에게 포인트를 지급하는 과정을 더 간단하고 편하게 만들고 싶다는 요청을 바탕으로 프로젝트를 시작했습니다.

설계 접근: 운영자가 고민 없이 사용할 수 있도록 입력 단계를 최소화하고, 포인트 지급·관리 흐름을 화면 기준으로 정리했습니다. 또한 운영 중 기준이 흔들리지 않도록 처리 규칙은 코드 로직으로 고정해 사용자 판단에 의존하지 않도록 구성했습니다.

운영 및 개선: 실제 매장에서 사용되며 전달된 피드백을 기준으로 입력 흐름과 화면 가독성을 개선하고, 업데이트를 지속적으로 반영했습니다.

운영 피드백 반영 사례 (요약)

  • 검색 UX: 전화번호 입력 후 엔터로 검색되길 원함 → 엔터/클릭 검색 모두 지원
  • 가독성: 전화번호 숫자 구분이 어려움 → 자동 하이픈 포맷 적용(입력/리스트 동일 기준)
  • 시인성: 주요 숫자 확인이 번거로움 → 크기·가중치 조정

※ 커뮤니케이션 원문 캡처 대신, 개선 항목 중심으로 정리했습니다.

기술/개념: Python(UI/로직) · 입력 단계 최소화 · 처리 규칙 고정(로직화) · 로컬 배포/업데이트 · 피드백 기반 UX 개선

2) 시스템 정보 수집 프로그램 (PC 사양 확인)

Tags: Delivered · Signed · Maintained

한 줄 요약: 환경별 조회 실패를 줄이기 위해 다중 조회 + fallback 구조를 적용한 PC 사양 확인 도구를 개발했습니다.

문제 정의: PC 환경별 API 지원 차이로 인해 단일 수집 방식에서는 시스템 정보(CPU/GPU/RAM 등) 누락 가능성이 존재했습니다.

설계 접근: 여러 수집 경로를 조합하고 실패 조건을 기준으로 대체 경로로 자동 전환되도록 fallback 구조를 설계했습니다. 예외 처리/로그를 포함해 유지보수 관점으로 구조화했습니다.

구현 및 결과: 다양한 환경에서 안정적으로 동작하는 형태로 배포했으며, 배포 신뢰성 강화를 위해 코드 서명을 적용했습니다. 보호 강화를 위해 난독화/워터마킹을 단계적으로 적용 중입니다.

기술/개념: Python · 다중 수집 전략 · failure handling(fallback) · 예외 처리/로그 · 배포 신뢰성(코드 서명)
