SYSTEM DESIGN · STABILITY · MAINTAINABILITY

실사용 업무 프로그램을 설계·납품·운영하는 백엔드/자동화 개발자

코인 세탁소 CRM, PC 사양 확인 프로그램을 유료 납품하고 운영 피드백 기반 업데이트를 수행해왔습니다.

실제 운영 환경에서 사용되는 프로그램을 납품·운영하며, 예외·실패 시나리오와 유지보수 관점을 고려한 설계를 경험했습니다.

기능 구현 자체보다 운영 편의성, 실패 대응, 수정 가능성을 우선합니다.

화면 구성도 시스템의 일부로 보고, 불필요한 요소를 제거한 미니멀 UI로 빠른 인지와 낮은 시각적 피로도를 목표로 합니다.

PROJECT 01 · 실사용 납품

1) 매장 운영 CRM 시스템 (포인트 관리)

한 줄 요약: 매장 운영자가 고객에게 포인트를 보다 편하게 지급할 수 있도록, 운영 편의성 개선 요청을 기준으로 CRM을 설계·구현했습니다.

배경(요청): 매장에서 고객 포인트 지급 과정이 보다 일관된 기준으로 관리되길 바라는 요청이 있었습니다.

설계 접근: 운영자가 고민 없이 사용할 수 있도록 입력 단계를 최소화하고, 포인트 지급·관리 흐름을 화면 기준으로 정리했습니다. 또한 운영 중 기준이 흔들리지 않도록 처리 규칙은 코드 로직으로 고정해 사용자 판단에 의존하지 않도록 구성했습니다.

운영 및 개선: 실제 매장에서 사용되며 전달된 피드백을 기준으로 입력 흐름과 화면 가독성을 개선하고, 업데이트를 지속적으로 반영했습니다.

UX 개선 사항 (실사용 기준)

  • 검색 UX 개선

    전화번호 입력 시 자동 검색 방식으로 개선. 기존 Enter/클릭 검색 요청에 대해 사용성 관점에서 자동 검색을 제안·적용했으며, 사용자 편의성이 크게 향상되었다는 긍정적인 피드백을 받음.

  • 가독성 개선
    • 전화번호 자동 하이픈 포맷 적용
    • 입력창과 결과 리스트에 동일한 포맷 기준 사용
  • 시인성 개선
    • 주요 숫자 정보의 폰트 크기 및 가중치 조정
    • 빠른 정보 인지를 목표로 UI 튜닝

※ 커뮤니케이션 원문 캡처 대신, 실사용 기준의 개선 항목 중심으로 정리했습니다.

기술/개념: Python(UI/로직) · 입력 단계 최소화 · 처리 규칙 고정(로직화) · 로컬 배포/업데이트 · 피드백 기반 UX 개선

PROJECT 02 · 실사용 납품

2) 시스템 정보 수집 프로그램 (PC 사양 확인)

한 줄 요약: 환경별 조회 실패를 줄이기 위해 다중 조회 + fallback 구조를 적용한 PC 사양 확인 도구를 개발했습니다.

문제 정의: PC 환경별 API 지원 차이로 인해 단일 수집 방식에서는 시스템 정보(CPU/GPU/RAM 등) 누락 가능성이 존재했습니다.

설계 접근: 여러 수집 경로를 조합하고 실패 조건을 기준으로 대체 경로로 자동 전환되도록 fallback 구조를 설계했습니다. 예외 처리/로그를 포함해 유지보수 관점으로 구조화했습니다.

구현 및 결과: 다양한 환경에서 안정적으로 동작하는 형태로 배포했으며, 배포 신뢰성 강화를 위해 코드 서명을 적용했습니다. 프로그램 실행 화면은 당사와 저작권 범위 협의가 진행 중이라 현재 비공개 처리했습니다. 보호 강화를 위해 난독화/워터마킹을 단계적으로 적용 중입니다.

기술/개념: Python · 다중 수집 전략 · failure handling(fallback) · 예외 처리/로그 · 배포 신뢰성(코드 서명)
