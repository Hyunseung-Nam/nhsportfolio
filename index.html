유료 납품 프로젝트 2건(코인 세탁소 CRM, PC 사양 확인 프로그램)을 실제 운영 환경에 적용했고, 현장 피드백을 반영해 개선을 이어가고 있습니다.

실사용 납품 프로젝트

아래 2개 프로젝트는 납품 후 실제 현장에서 운영된 사례입니다.

1) 코인 세탁소 고객 관리 프로그램 (CRM)

한 줄 요약: 수기 포인트 관리에서 발생하던 오류와 누락을 줄이기 위해 매장 운영형 CRM을 구축했습니다.

문제: 중소 매장에서 고객 포인트를 수기로 관리해 계산 실수, 누락, 기준 불일치로 운영 부담이 발생했습니다.

해결: 운영자가 최소 입력으로 포인트를 지급·관리하도록 흐름을 단순화하고, 데이터 처리 기준을 로직으로 고정했습니다.

결과: 실제 매장에서 사용 중이며, 현장 사용자 피드백을 반영한 UX 업데이트를 지속적으로 적용했습니다.

기술: Python (UI 및 로직 구현), 데이터 처리 로직, 로컬 배포/업데이트 관리, 사용자 피드백 기반 UX 개선

2) PC 사양 확인 프로그램

한 줄 요약: 환경별 조회 실패를 줄이기 위해 다중 조회 + fallback 구조를 적용한 사양 확인 도구를 개발했습니다.

문제: PC 환경별 차이로 단일 방식 조회만으로는 CPU/RAM 등 시스템 정보 수집의 안정성이 떨어질 수 있었습니다.

해결: 여러 조회 방식을 조합하고 실패 시 대체 경로로 전환되도록 설계했습니다. 예외 처리와 유지보수 관점의 구조로 구현했습니다.

결과: 다양한 환경에서 안정적으로 동작하는 형태로 배포했으며, 코드 서명을 적용했습니다. 배포 안정성 강화를 위해 난독화/워터마킹을 적용 중입니다.

기술: Python, 시스템 정보 수집 로직, 다중 조회·fallback 설계, 배포 환경 고려(코드 서명), 유지보수 대응

문제 해결 방식 / 강점

비개발자 대상 UX

입력 단순화

현장 사용자가 기능보다 흐름을 먼저 이해하도록 입력 단계와 용어를 최소화합니다.

운영 안정성

실패 시나리오 설계

환경 차이·예외 케이스를 전제로 설계하고, 실패 시 대체 경로로 전환되도록 구성합니다.

유지보수 관점

재현 후 수정

이슈는 바로 고치기보다 재현 조건을 먼저 좁힌 뒤, 수정 범위를 정하고 배포합니다.

개인 프로젝트 (업무 흐름 개선 중심)

아래 항목은 납품 프로젝트와 구분되는 개인 프로젝트입니다. 실제 문제(보고 누락, 수기 정리)를 줄이기 위한 흐름 설계에 집중했습니다.

소규모 ERP (로그인 연동)

한 줄 요약: 보고 누락과 수기 정리로 인한 비효율을 줄이기 위해 로그인 기반 소규모 ERP를 설계·구현했습니다.

문제: 업무 보고가 여러 채널로 흩어져 누락/중복이 발생했고, 정리 작업에 시간이 많이 소요되었습니다.

해결: 로그인 기반 사용자 구분 구조를 구성하고, 입력·저장·조회 흐름을 일관되게 만들어 보고 상태를 한 곳에서 관리하도록 했습니다.

결과: 보고 누락 가능성을 줄이고, 반복 정리 시간을 절감해 더 중요한 업무에 시간을 쓰도록 했습니다.

기술: Python, 인증/로그인 로직, CRUD 흐름 설계, 데이터 관리 구조, 화면 흐름 구성

기술 스택 (활용 맥락)

Python

납품 프로젝트와 ERP에서 UI/로직 구현, 예외 처리, 데이터 처리 기준 고정, 운영 피드백 반영에 사용했습니다.

업무 스타일

  • 요청을 받을 때: “무엇을 만들까”보다 “왜 필요한가 / 어디서 문제가 나는가”를 먼저 정리합니다.
  • 변경이 생길 때: 영향 범위를 먼저 확인하고, 가능한 선택지(그대로/대안/추가 작업)를 함께 공유합니다.
