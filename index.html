실사용 납품 프로젝트

아래 2개 프로젝트는 유료 납품 후 실제 현장에서 운영된 사례입니다.

1) 코인 세탁소 고객 관리 프로그램 (CRM)

한 줄 요약: 수기 포인트 관리에서 발생하던 오류와 누락을 줄이기 위해 매장 운영형 CRM을 구축했습니다.

문제: 중소 매장에서 고객 포인트를 수기로 관리해 계산 실수, 누락, 기준 불일치로 운영 부담이 발생했습니다.

해결: 매장 운영자가 간단한 입력만으로 포인트를 지급·관리하도록 입력 흐름을 단순화하고, 데이터 처리 기준을 프로그램 로직으로 고정했습니다.

결과: 실제 매장에서 사용 중이며, 현장 사용자 피드백을 반영한 UX 업데이트를 지속적으로 적용했습니다.

기술: Java, HTML, 데이터 처리 로직, 배포/업데이트 관리

2) PC 사양 확인 프로그램

한 줄 요약: 환경별 조회 실패를 줄이기 위해 다중 조회 + fallback 구조를 적용한 사양 확인 도구를 개발했습니다.

문제: PC 환경별 차이로 단일 방식 조회만으로는 CPU/RAM 등 사양 확인 정확도가 떨어질 수 있었습니다.

해결: 여러 조회 방식을 조합하고 실패 시 대체 경로로 전환되도록 설계해 조회 신뢰도를 높였습니다. 오류 처리와 유지보수를 고려한 구조로 구현했습니다.

결과: 범용 환경에서 안정적으로 사용 가능한 형태로 납품했으며, 업데이트와 유지보수를 이어가고 있습니다. 코드 서명 적용을 완료했고 난독화/워터마킹을 진행 중입니다.

기술: Python, 시스템 정보 수집 로직, 배포 환경 고려(코드 서명), 유지보수 대응

문제 해결 방식 / 강점

비개발자 대상 UX

입력 단순화

현장 사용자가 기능보다 흐름을 먼저 이해하도록 입력 단계와 용어를 최소화합니다.

운영 안정성

오류/유지보수 고려

기능 구현 단계에서 실패 시나리오와 수정 가능성을 함께 설계해 운영 리스크를 줄입니다.

요구사항 처리

구조화 후 구현

요청 사항을 기능 단위로 분해하고 우선순위와 영향 범위를 정리한 뒤 구현합니다.

개인 프로젝트 (실사용 전제 설계)

아래 항목은 납품 프로젝트와 구분되는 개인 프로젝트이며, 실제 사용 가능성을 전제로 설계했습니다.

업무 자동화 스크립트

한 줄 요약: 반복 업무를 줄이기 위한 자동화 흐름을 스크립트 형태로 구현했습니다.

문제: 수작업 반복이 많아 처리 시간과 오류 가능성이 높았습니다.

해결: 입력-처리-결과 단계를 명확히 분리해 재사용 가능한 실행 구조로 구성했습니다.

결과: 반복 업무 시간을 줄이고 수정 가능한 구조로 관리하고 있습니다.

기술: Python, 자동화 로직, 실행 로그 관리

기술 스택 (활용 맥락)

Python

PC 사양 확인 로직과 자동화 스크립트에서 데이터 수집, 예외 처리, 실행 흐름 구성에 사용했습니다.

Java

코인 세탁소 CRM의 핵심 비즈니스 로직(포인트 지급/차감 기준) 구현에 사용했습니다.

HTML

비개발자 사용자 기준으로 화면 구조를 단순화하고 입력 실수를 줄이는 UI 뼈대를 구성했습니다.

Git / 배포 관리

버전 이력 관리, 업데이트 반영, 운영 중 수정 배포 흐름 정리에 사용했습니다.

  • 커뮤니케이션 기준: 변경 요청 시 영향 범위, 일정, 대안 3가지를 함께 공유합니다.
  • 범위 인식: 초기 납품 범위와 후속 유지보수 범위를 분리해 운영 혼선을 줄입니다.
  • 유지보수 관점: 운영 중 발생한 이슈는 재현 조건을 명확히 만든 뒤 수정·배포합니다.
