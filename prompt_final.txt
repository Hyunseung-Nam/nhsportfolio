다음 조건을 만족하도록 포트폴리오 HTML에 이미지 섹션을 구성해줘.

[공통 조건]
- 모든 이미지 파일은 assets/ 폴더에 이미 존재함
- 이미지 경로는 상대 경로 assets/파일명 형태로 사용
- before사진: before.png
- after사진: after.png
- 로그창 사진: log.png
- 코드서명 사진: code_sign.png
- 이미지 파일명은 임의로 생성하지 말 것
- 불필요한 장식 없이, 설명 보조 자료 역할로만 배치
- 이미지는 텍스트를 대체하지 않고 이해를 돕는 용도로만 사용
- 마케팅성 표현, 과장된 성과 강조 금지

────────────────
[프로젝트 1: 매장 운영 CRM 시스템]

사용 가능한 이미지:
1) CRM UX 개선 전 화면 (Before)
2) CRM UX 개선 후 화면 (After)
3) CRM 로그 출력 화면 (운영/이슈 추적용)

요구사항:
- Before / After 화면은 나란히 비교 배치
- 로그 화면은 비교 영역 아래 또는 별도 블록으로 배치
- 로그 화면의 역할이 명확히 드러나도록 설명 추가:
  · 입력/처리 흐름 확인
  · 운영 중 이슈 재현 및 추적 목적
- 아래 맥락이 자연스럽게 연결되도록 구성:
  · 사용자 UX 개선 → 운영 안정성 확보 → 로그 기반 관리
- 로그 화면은 “디버깅용”이 아닌 “운영 보조 수단”으로 표현

────────────────
[프로젝트 2: PC 사양 확인 프로그램]

사용 가능한 이미지:
4) 코드 서명 확인 화면 (Windows 디지털 서명)

요구사항:
- 코드 서명 화면:
  · 배포 환경에서의 신뢰성 확보 목적 강조
  · 보안 과장 표현 금지 (운영 신뢰성 관점 유지)
- Mermaid 다이어그램을 보조하는 시각 자료로 배치

────────────────
[설명 문구 스타일 가이드]

- 설명형·사실 중심 문장 사용
- 예시 톤:
  · “실제 운영 화면 예시”
  · “UX 개선 적용 후 화면”
  · “운영 중 이슈 추적을 위한 로그 출력 예시”
  · “배포 신뢰성 확보를 위한 코드 서명 적용 화면”
- 감정적 표현, 과도한 자기 평가 표현 금지

────────────────
[주의사항]

- 기존 텍스트 구조(헤딩, 문단 흐름)는 유지
- 이미지 배치로 레이아웃이 과도하게 늘어나지 않도록 조정
- 이미지가 없더라도 텍스트만으로 이해 가능한 구조 유지
