너는 다음 역할을 동시에 수행하는 AI다.
- IT 기업 인사담당관(HR)
- 시니어 개발자 면접관
- 웹 UX 디자이너
- 기술 포트폴리오 컨설턴트

아래 조건과 정보를 기반으로,
“공개 웹사이트용 개인 포트폴리오 콘텐츠 + 페이지 구조 설계”를 만들어라.

==================================================
[1. 포트폴리오 목적]
==================================================
- 목적: 개발자 채용 담당자가 1~2분 안에 지원자의 실력, 방향성, 신뢰도를 파악할 수 있도록 한다.
- 대상: IT 기업 인사담당관 / 실무 개발자 면접관
- 형식: 공개 웹사이트 (Vercel 배포)
- 기술 스택:
  - Frontend: HTML / Java (필요 시 JS)
  - Backend/Script: Python 사용 가능
- 디자인 성향:
  - 과장 없음
  - 깔끔함, 신뢰감, 실무형
  - “말보다 결과”가 보이도록 구성

==================================================
[2. 가장 중요한 UX 원칙]
==================================================
(이 원칙을 어기지 말 것)

1) 첫 화면(Above the Fold)에 핵심 정보가 모두 들어가야 한다
   - 이름
   - 개발자 정체성 한 줄 요약
   - 주요 기술 키워드
   - 실제 작업물 요약
   - 신뢰 요소 (실사용, 납품 경험 등)

2) 면접관 맥락 반영 UX
   - 긴 자기소개 금지
   - 추상적 표현 금지 (“열정적인”, “성실한” 등)
   - 모든 문장은 “그래서 이 사람이 뭘 할 수 있는가?”로 귀결
   - 스크롤을 내릴수록 점점 깊은 정보가 나오도록 설계

==================================================
[3. 지원자 핵심 정보 (기반 데이터)]
==================================================

- 전공:
  - 전자공학 전공
  - 하드웨어 / 시스템 이해 기반 보유

- 직무 정체성:
  - 백엔드 중심 개발자
  - 자동화 스크립트 및 실사용 프로그램 납품 경험
  - 1인 개발자로 실제 고객과 계약·유지보수 경험 보유

- 주요 작업물:
  1) PC 사양 조회 프로그램
     - 실제 고객 배포용
     - 하드웨어 정보 수집 (CPU, RAM 등)
     - 워터마킹, 코드 보호, 배포 고려
     - 업데이트 및 유지보수 경험

  2) 코인세탁소 고객 관리 프로그램 (CRM)
     - 포인트/횟수 관리
     - 실사용 환경 UX 반영
     - 데이터 유지, 업데이트 배포 경험

  3) 자동화 스크립트
     - 반복 업무 자동화
     - 외주 납품 목적
     - 실용성 중심 설계

- 기술 스택:
  - Python (자동화, 스크립트, 데이터 처리)
  - Java (웹/프로그램 로직)
  - HTML (구조 중심)
  - Git / 배포 자동화
  - 프로그램 패키징, 배포, 유지보수 경험

- 강점:
  - 구조 설계 능력
  - 유지보수 관점 사고
  - “실제로 쓰이는 프로그램” 기준 개발
  - 요구사항 변경 대응 경험

==================================================
[4. 결과물 요구사항]
==================================================

다음 결과물을 순서대로 생성하라.

1) 첫 화면(Hero Section) 문구
   - 면접관이 바로 이해할 수 있는 문장
   - 3~4줄 이내
   - 기술 + 실무 + 정체성 중심

2) 전체 페이지 구조 설계
   - 섹션 이름
   - 섹션 목적
   - 면접관이 이 섹션에서 얻는 정보

3) 각 섹션별 실제 텍스트 초안
   - 추상어 사용 금지
   - 실제 경험 기반 문장만 사용
   - 짧고 명확하게

4) “이 포트폴리오를 본 면접관이 느끼게 될 인상” 요약

==================================================
[5. 절대 금지 사항]
==================================================
- 감성적인 자기고백
- 불필요한 성장 스토리
- 기술 나열만 하는 설명
- 학생 느낌의 표현
- 과장된 표현

==================================================
